# /ss-checklist

执行质量门检查，确保变更满足质量标准。

## 语法

```
/ss-checklist
```

## 适用模式

两种模式皆有。标准模式：proposal 后 /10。增强模式：spec 后 /25。在 /ss-create 流程中自动调用，也可手动调用。

## AI 行为

执行此命令后，AI 会：

1. 读取 checklist.md
2. 逐项检查代码质量
3. 报告检查结果
4. 标记通过/未通过项

## 检查内容

- 代码质量
- 测试覆盖
- 安全检查
- 性能要求
- 文档完整性

## 示例输出

```
◆ 质量检查

代码质量
  ✓ 代码符合编码规范
  ✓ 无 lint 错误
  ✓ 无类型错误

测试
  ⚠ 单元测试覆盖率 65%（目标 80%）
  ✓ 集成测试通过

安全
  ✓ 无硬编码凭证
  ✓ 输入已验证

建议：增加单元测试覆盖率
```

## 后续步骤

检查通过后：

```
/ss-apply    # 继续执行任务
```
