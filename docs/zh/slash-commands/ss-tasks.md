# /ss-tasks

从 proposal 生成分阶段的任务清单。

## 语法

```
/ss-tasks
```

## 参数

此命令没有参数，会自动检测当前活跃的变更。

## AI 行为

执行此命令后，AI 会：

1. 读取当前变更的 proposal.md
2. 如果是增强模式，还会读取 spec.md
3. 分析需求，生成分阶段的任务清单
4. 写入 tasks.md
5. 等待你确认任务列表

## 生成的任务特点

- 每个任务 < 1 小时工作量
- 按依赖关系排序
- 标记可并行任务 `[P]`
- 关联验收标准（增强模式）

## 示例输出

```markdown
## 阶段 1: 基础设施

### 任务 1.1: 创建数据库模型
- 创建 User 模型
- 添加必要字段

### 任务 1.2: 创建 API 路由 [P]
- 创建 /auth/login 路由
- 创建 /auth/register 路由

## 阶段 2: 业务逻辑

### 任务 2.1: 实现认证逻辑
...
```

## 后续步骤

任务生成后，继续使用：

```
/ss-apply    # 执行任务
```
